# Reference Implementation (Еталонна Реалізація)

Це фінальний код проєкту "Rock-Paper-Scissors MMO", до якого ви маєте прийти в кінці курсу.

⚠️ **Попередження:** Не копіюйте цей код сліпо. Ви використовуєте його для самоперевірки (*"Як архітектор вирішив цю проблему?"*), а не для здачі. Я побачу різницю.

## Структура проєкту (Best Practices)

Ми використовуємо принцип **Separation of Concerns** (Розділення відповідальності).

```text
src/
├── assets/              # Статичні файли (картинки, іконки, звуки)
│   ├── css/
│   │   ├── main.css     # Глобальні стилі (скидання, шрифти)
│   │   └── game.css     # Стилі специфічні для гри
│   └── images/
│       └── logo.png
│
├── js/                  # Логіка (JavaScript)
│   ├── app.js           # Точка входу (Main Thread, ініціалізація)
│   ├── game/            # Ігрова логіка
│   │   ├── engine.js    # Game Loop, фізика
│   │   └── ai.js        # Логіка бота (Markov Chains)
│   ├── data/            # Робота з даними
│   │   ├── storage.js   # Обгортка над LocalStorage
│   │   └── api.js       # Мережеві запити (Fetch)
│   └── utils/           # Допоміжні функції
│       └── security.js  # Хешування паролів
│
└── index.html           # Єдиний HTML файл (SPA)
```

## Як це запустити?

Цей код використовує ES6 Modules (`import/export`), тому він **не працюватиме**, якщо відкрити `index.html` просто як файл.
Вам потрібен локальний сервер:

1.  Відкрийте цю папку в VS Code.
2.  Натисніть "Go Live" (Live Server).
