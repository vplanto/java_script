# Лекція 2: CSS як система координат

## Що таке CSS?

**CSS (Cascading Style Sheets)** — мова опису зовнішнього вигляду HTML-документа. Якщо HTML — це кістяк (структура та зміст), то CSS — це зовнішність: кольори, розміри, відступи, шрифти, анімації.

```
HTML  →  ЩО відображати   (<p>, <div>, <img>)
CSS   →  ЯК відображати   (колір, розмір, позиція)
JS    →  ЩО робити        (логіка, взаємодія)
```

CSS працює як набір **правил**: ви вибираєте елемент (*селектор*) і задаєте йому властивості (*declarations*):

```css
/* селектор { властивість: значення; } */
p {
    color: red;
    font-size: 18px;
}
```

Слово **«Cascading»** (каскадні) — ключове. Якщо кілька правил описують один елемент, перемагає те, що **конкретніше** або написане **пізніше**. Саме тому `z-index: 9999` не завжди перемагає — каскад складніший, ніж здається.

Відкрийте [durgasoft.com](https://www.durgasoft.com/) — один з найпопулярніших сайтів з курсами Java і Python в Індії. Мільйони студентів навчились за цими матеріалами. Але подивіться на нього: таблиці, яскраві кольори, тексту більше ніж на сторінці Wikipedia, і жодного сенсу в розташуванні елементів.

**2007 рік.** iOS і перший Safari на мобільних. Раптом виявляється, що сайти з таблицями і фіксованою шириною у 800px на телефоні виглядають жахливо. Починається нова ера.

### Netscape vs IE: Браузерні війни та народження «CSS Hacks»

У 1990-х Netscape і Internet Explorer змагались, хто додасть більше нестандартних тегів. Netscape придумав `<blink>` (миготливий текст), IE — `<marquee>` (рухомий текст). Обидва знали, що стандарту ще немає, тому творили хаос.

Розробники писали різний код для різних браузерів:
```css
/* Хак для IE6: */
* html .box { width: 100px; }

/* Нормальний CSS: */
.box { width: 120px; }
```

Цей період отримав назву **«Браузерні війни»**. Він закінчився перемогою IE (95% ринку у 2002 р.) — і це було катастрофою: IE зупинився в розвитку, стандарти не розвивались, і веб застряг у болоті несумісностей на роки.

**Саме тому** `box-sizing: border-box` не є дефолтом у CSS до сьогодні — міняти дефолт означає зламати мільярди старих сайтів. CSS несе спадщину тих часів у кожній своїй дефолтній властивості.

    * а) 100px
    * б) 110px
    * в) 120px
2.  Ви поставили елементу `z-index: 9999`, але він все одно знаходиться ПІД іншим елементом. Чому?

<details markdown="1">
<summary>Відповіді (для інженера)</summary>

1.  **120px** (за замовчуванням). Стандартна модель (`content-box`) додає рамки та відступи ДО ширини. Це пекло для розрахунків. Інженери використовують `border-box`.
2.  **Stacking Context (Контекст накладання).** `z-index` працює тільки всередині свого контексту. Якщо батько вашого елемента має нижчий `z-index`, ніж сусід, то ніякі `9999` не допоможуть дитині "вистрибнути" вище.

## 1. Box Model: Математика інтерфейсу

Кожен елемент у DOM — це прямокутник. Навіть якщо він круглий (через `border-radius`), для браузера це прямокутна коробка.

### Інженерна проблема:
У стандартному CSS (`box-sizing: content-box`), якщо ви кажете `width: 100px` і `padding: 20px`, реальна ширина стає **140px**.
Це ламає сітки. Ви розраховуєте на 100, отримуєте 140, верстка "їде".

### Рішення: `border-box`
```css
* {
    box-sizing: border-box;
}
```

Тепер `width: 100px` означає, що весь елемент (разом з рамками і відступами) займе рівно 100px. Браузер сам стисне контент.
**Це перше, що ви пишете в будь-якому CSS файлі.**

### Відступи в CSS: зовні і всередині коробки

Найчастіша плутанина для новачків — різниця між `margin` і `padding`. Обидва про відступи, але між різними речами:

```
┌─────────────────────────────────────┐
│             MARGIN (зовні)          │  ← відступ від сусідніх елементів
│  ┌───────────────────────────────┐  │
│  │         BORDER                │  │  ← рамка
│  │  ┌─────────────────────────┐  │  │
│  │  │      PADDING (всередині) │  │  │  ← відступ від краю до змісту
│  │  │  ┌───────────────────┐  │  │  │
│  │  │  │   CONTENT (текст) │  │  │  │
│  │  │  └───────────────────┘  │  │  │
│  │  └─────────────────────────┘  │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

| Властивість | Де діє | Аналогія |
|---|---|---|
| `padding` | Між краєм елемента і його вмістом | Пінопласт в коробці |
| `border` | Сама коробка | Стінки коробки |
| `margin` | Між елементом і сусідами | Відстань між коробками на полиці |

```css
.card {
    padding: 16px;     /* внутрішній відступ від краю картки до тексту */
    border: 1px solid #ccc;
    margin: 24px;      /* зовнішній відступ: картка відступає від інших елементів */
}
```

> ⚠️ **Margin Collapse** — сюрприз CSS: коли два вертикальні `margin` зустрічаються (два сусідні блоки або батько і перша дитина), вони **не складаються, а зливаються** — береться більший. `margin-top: 30px` на першому елементі всередині `div` може «протекти» назовні контейнера. Це не баг — це спадщина типографіки 1960-х. Виправляється через `padding-top: 1px` або `overflow: hidden` на батьку.

## 2\. Positioning: Керування реальністю

За замовчуванням елементи йдуть один за одним зверху вниз — це **Normal Flow**. `position` дозволяє вирвати елемент з цього потоку і розмістити його куди завгодно.

| Значення | У потоці? | Координати відносно | Коли використовувати |
|---|:---:|---|---|
| `static` | ✅ | — (дефолт, без позиціонування) | Завжди за замовчуванням |
| `relative` | ✅ | Своєї початкової позиції | Зсув елемента без впливу на сусідів; батько для `absolute` |
| `absolute` | ❌ | Найближчого батька з `position` ≠ `static` | Tooltip, badge, спливаючі меню над контентом |
| `fixed` | ❌ | Вікна браузера (viewport) | Sticky header, кнопка «↑ нагору», cookie-банер |
| `sticky` | ✅/❌ | Потоку → потім viewport при скролі | Заголовки таблиць, розділів, що «прилипають» |

### Приклади коду

**`relative` — зсунути, але місце зберегти:**
```css
.icon {
    position: relative;
    top: -4px;  /* підняти на 4px, але сусіди цього не відчують */
}
```

**`absolute` — «вирвати» і розмістити точно:**
```css
/* Батько ОБОВ'ЯЗКОВО має position: relative */
.card {
    position: relative;
}
.badge {
    position: absolute;
    top: 8px;
    right: 8px;  /* кут картки */
}
```
> ⚠️ **Найпоширеніша помилка:** забути поставити `position: relative` на батька. Тоді `absolute`-елемент «злітає» і позиціонується відносно всього `<body>`.

**`fixed` — приклеїти до екрану:**
```css
.header {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;  /* розтягнути на всю ширину вікна */
    z-index: 100; /* поверх решти контенту */
}
```

**`sticky` — гібрид: спочатку в потоці, потім прилипає:**
```css
.section-title {
    position: sticky;
    top: 0;  /* прилипає до верху, коли доскролив до нього */
}
```

## 3\. Flexbox та Grid: Розташування елементів

**Задача:** як розташувати кілька блоків поруч — наприклад, три картки товару в ряд?

Здається просто, але це одна з найстаріших проблем CSS. Уявіть: `<div>` — це блоковий елемент, він за замовчуванням займає **весь рядок** і штовхає наступний вниз. Як змусити їх стати поряд?

### Як це робили до 2015 року (і чому це було погано)

До появи Flexbox розробники використовували `float` — властивість, яка спочатку призначалась для **обтікання картинок текстом**. Але більше нічого в CSS не було, тому її «зламали» для верстки:

```css
/* Стара верстка через float — не робіть так */
.card {
    float: left;
    width: 33%;
}
.container::after {
    content: "";
    display: block;
    clear: both; /* «магічне» очищення, без якого контейнер схлопується */
}
```

Це працювало, але зламувалось при будь-якому нестандартному контенті. Центрування по вертикалі через `float` — взагалі неможливе.

### Сучасне рішення: Flexbox і Grid

**Ключовий принцип:** замість того щоб «хакати» поведінку елементів — ви **декларуєте намір** батькові:

```css
.container {
    display: flex;  /* «діти, станьте в ряд» */
}
```

| Інструмент | Вимір | Задача | Приклад |
|---|:---:|---|---|
| **Flexbox** | 1D (рядок або колонка) | Вирівняти елементи по одній осі | Меню, рядок карток, кнопки у формі |
| **Grid** | 2D (рядки + колонки) | Побудувати повноцінну сітку сторінки | Галерея, layout сторінки, дашборд |

### Flexbox — вирівняти елементи в рядок

```css
.menu {
    display: flex;
    justify-content: space-between; /* розподілити по горизонталі */
    align-items: center;            /* вирівняти по центру вертикально */
    gap: 16px;                      /* відступи між елементами */
}
```

> Три рядки CSS, і ви отримуєте те, на що раніше витрачали 20 рядків `float`-хаків.

### Grid — побудувати сітку

```css
.gallery {
    display: grid;
    grid-template-columns: repeat(3, 1fr); /* три рівні колонки */
    gap: 24px;
}
```

`fr` — **fraction** (частка вільного простору). `repeat(3, 1fr)` — три рівні колонки, які автоматично підлаштовуються під ширину контейнера.

### Коли що використовувати?

```
Потрібно розташувати елементи В РЯДОК або КОЛОНКУ?  →  Flexbox
Потрібна СІТКА (рядки І колонки одночасно)?         →  Grid
Потрібно розмістити елемент ТОЧНО на сторінці?      →  position: absolute/fixed
```

> **Правило:** не використовуйте JS (`element.style.left = ...`) там, де справляється CSS. JS — для логіки, CSS — для геометрії. Кожен `element.style` у циклі — це потенційний Layout Thrashing (з минулої лекції).

## 4\. Селектори: мова запитів до DOM-дерева

Селектори CSS — це **повноцінна мова пошуку** по HTML-дереву. Так само як `querySelector` в JavaScript дозволяє знайти елемент у DOM, CSS-селектор каже браузеру: *«знайди саме ці вузли і застосуй до них стиль»*.

### Базові типи

| Селектор | Синтаксис | Що знаходить | Вага |
| :--- | :--- | :--- | :--- |
| Тег | `p` | Усі `<p>` на сторінці | 1 |
| Клас | `.btn` | Усі елементи з `class="btn"` | 10 |
| ID | `#nav` | Єдиний елемент з `id="nav"` | 100 |
| Inline | `style="..."` | Напряму на елементі | 1000 |
| `!important` | `color: red !important` | Перекриває все | ∞ |

Крім базових є **комбінатори** (навігація по дереву: `div > p`, `h2 + p`) та **псевдокласи** (стан елемента: `:hover`, `:focus`, `:nth-child`) — це окрема тема наступних лекцій.

### Специфічність — хто перемагає?

Якщо два правила описують один елемент, перемагає **конкретніше** (вища вага). `ID` перекриває клас, клас перекриває тег.

> **Практика:** уникайте `ID` у стилях і `!important` — це гонка озброєнь, яка закінчується нечитабельним кодом. Будуйте стилі через класи.

## 5\. Межі CSS: що може і чого не може

Одне з ключових розумінь для розробника — **де CSS закінчується і де починається JS**.

| CSS може | JS потрібен |
|---|---|
| Змінити колір/розмір/відступ при `:hover` | Змінити стиль залежно від **даних** (значення з сервера) |
| Плавна анімація (`transition`, `@keyframes`) | Анімація, яка залежить від **логіки** (таймер, стан гри) |
| Показати/сховати елемент через клас | Додати/видалити клас **динамічно** у відповідь на подію |
| Розташувати елементи в сітку | Змінити координати елемента **програмно** (`top`, `left`) |
| Стилізувати форму | Валідувати дані форми, відправляти запити |

```
CSS:  ЯКИМ виглядає елемент зараз
JS:   КОЛИ і ЧОМУ він змінює вигляд
```

> **Наш проєкт (кнопка-втікач):** CSS задає початкову позицію кнопки (`position: absolute`, `top`, `left`). JS змінює ці значення при події `mouseover`. Це і є правильний розподіл відповідальності.

## Практикум: "CSS Хірургія"

Відкрийте DevTools на будь-якому сайті.

**Відкрийте будь-який сайт у Chrome/Firefox.** Будемо «препарувати» його CSS прямо у браузері.

---

### Завдання 1: Відкрити DevTools та знайти елемент

1. Відкрийте будь-який сайт (наприклад, [wikipedia.org](https://wikipedia.org))
2. Натисніть **F12** або **Ctrl+Shift+I** (Mac: **Cmd+Option+I**) — відкриється панель DevTools
3. Клікніть на іконку **курсор** (⬚↖) у лівому верхньому куті DevTools — або натисніть **Ctrl+Shift+C**
4. Наведіть мишу на будь-який заголовок на сторінці — він підсвітиться синім
5. Клікніть на нього — у панелі **Elements** відкриється відповідний HTML-тег

> Ви щойно знайшли вузол DOM. Праворуч у вкладці **Styles** видно всі CSS-правила, що застосовуються до нього.

---

### Завдання 2: Зламати Layout (position: absolute)

1. У панелі **Elements** виберіть будь-який `<div>` або `<header>`
2. У вкладці **Styles** (праворуч) клікніть у порожнє місце — з'явиться поле вводу
3. Введіть: `position: absolute` та натисніть Enter
4. Подивіться на сторінку — елемент «вирвався» з потоку, сторінка «схлопнулась» під ним

> Ось так виглядає `absolute` без `relative`-батька. Елемент полетів у куток `<body>`.

5. Спробуйте додати: `top: 50px; left: 200px;` — елемент переміщується точно за координатами

---

### Завдання 3: Debug Mode — показати всі «коробки»

1. Відкрийте вкладку **Console** (внизу DevTools або Ctrl+\`)
2. Вставте цей рядок і натисніть Enter:

```javascript
document.head.insertAdjacentHTML('beforeend', '<style>* { outline: 1px solid red !important; }</style>')
```

3. Тепер кожен елемент на сторінці обведений червоною рамкою — ви бачите справжню Box Model кожного вузла

> Зверніть увагу: `outline` не впливає на Layout (на відміну від `border`) — він малюється поверх, не зсуваючи сусідів.

4. Щоб прибрати — просто оновіть сторінку (F5)

---

### Завдання 4: Знайти звідки береться стиль

1. Виберіть будь-який елемент через інструмент курсора
2. У Styles подивіться на список правил — вони відсортовані за **специфічністю**
3. Перекреслені правила — **перекриті** більш конкретним селектором
4. Наведіть мишу на назву файлу (наприклад, `styles.css:142`) — побачите точний рядок у вихідному файлі

> Це і є «детектив специфічності» — ви бачите, чому якийсь стиль не працює.

## Домашнє завдання: Підготовка до Bad UI

Для виконання наступного воркшопу вам треба вміти рухати елементи через CSS.

**Мета:** зробити кнопку-«втікача», яка лежить в абсолютних координатах на сторінці. На наступному занятті ми навчимо її тікати від курсора мишки через JavaScript.

1. Створіть файл `index.html` у корені проєкту.
2. Створіть файл `style.css` поруч з ним.
3. Підключіть CSS до HTML через `<link>`.
4. Зробіть кнопку **«Натисни мене»** з класом `escape-btn`:
   - Дайте їй `position: absolute`
   - Встановіть початкові координати `top` і `left` (наприклад, `200px` і `300px`)
   - Додайте будь-який колір, padding, border-radius — щоб виглядало як кнопка
5. Відкрийте файл у браузері і змініть координати кнопки через **DevTools**:
   - Натисніть **F12** → вкладка **Elements**
   - Натисніть **Ctrl+Shift+C** і клікніть на кнопку — вона підсвітиться в Elements
   - У правій панелі **Styles** знайдіть `.escape-btn` → рядки `top: 200px` і `left: 300px`
   - Клікніть на число `200` — з'явиться текстовий курсор, введіть нове значення
   - **Швидкий спосіб:** клікніть на число і натискайте **↑/↓** — значення змінюється на 1px; з **Shift+↑** — на 10px одразу
   - Спостерігайте: кнопка рухається по екрану в реальному часі без перезавантаження сторінки

## Контрольні питання

1. У чому різниця між `box-sizing: content-box` і `box-sizing: border-box`? Яке з них є кращою практикою і чому?

<details>
<summary>Відповідь</summary>

`content-box` (стандарт за замовчуванням): `width` враховує тільки контент, тому реальна ширина = `width + padding + border`. Це ламає розрахунки при верстці сіток.

`border-box`: `width` включає padding і border — ширина елемента залишається рівно такою, як ви вказали. Ставте `* { box-sizing: border-box; }` першим рядком у будь-якому CSS файлі.

</details>

2. У вас є два `div` один під одним. У першого `margin-bottom: 20px`, у другого `margin-top: 30px`. Який буде відступ між ними?

<details>
<summary>Відповідь</summary>

**30px**, а не 50px. Це Margin Collapse: вертикальні зовнішні відступи суміжних блоків «зливаються» в один — береться максимальне значення. Це поведінка, успадкована з типографіки.

</details>

3. Елемент має `z-index: 9999`, але він все одно знаходиться під сусіднім елементом. Яка найімовірніша причина?

<details>
<summary>Відповідь</summary>

Batько цього елемента утворює **Stacking Context** з нижчим `z-index`. `z-index` працює лише всередині свого контексту — він не виштовхує елемент «назовні». Якщо батько знаходиться нижче сусіда батьківського рівня, дитина з будь-яким `z-index` не допоможе.

</details>

4. Чому краще використовувати **CSS Grid** для розкладки сторінки, а не обчислювати позиції через JavaScript?

<details>
<summary>Відповідь</summary>

JavaScript-розрахунки позицій (`element.style.left = ...`) викликають **Layout Thrashing** — браузер змушений перераховувати геометрію при кожній зміні. CSS Grid — декларативний: ви описуєте структуру, браузер сам розраховує позиції один раз. Простіший код, краша продуктивність.

</details>

5. Поясніть різницю між `position: absolute` і `position: fixed`. У якому випадку використовуєте кожен?

<details>
<summary>Відповідь</summary>

`absolute` — координати відносно найближчого позиціонованого предка (`relative`/`absolute`/`fixed`). Прокручується разом зі сторінкою. Використання: елементи всередині компонента (tooltip, badge).

`fixed` — координати відносно вікна браузера. Не рухається при прокручуванні. Використання: sticky хедери, модальні вікна, кнопка «нагору».

</details>
