# Налаштування інженерного середовища (Environment Setup)

Ви не зможете будувати хмарочос, маючи лише молоток. Вам потрібен професійний інструментарій.
Це **обов'язковий набір** для проходження курсу.

**Контекст:** Ви переходите з C++ (компільована мова) на JavaScript (інтерпретована мова).
Ваш звичний ланцюжок `Write -> Compile -> Link -> Run` змінюється, але фундаментальні інструменти залишаються тими ж.

---

## 1. The Editor: Visual Studio Code

Ми не пишемо код у Word або Notepad. Ми використовуємо IDE.
Стандартом індустрії є **Visual Studio Code**.

1.  **Завантажити:** [code.visualstudio.com](https://code.visualstudio.com/)
2.  **Встановити розширення (Extensions):**
    *Відкрийте вкладку Extensions (`Ctrl+Shift+X`) та встановіть:*
    * **Prettier - Code formatter:** Аналог `clang-format`. Щоб ваш код виглядав охайно автоматично.
    * **ESLint:** Аналог статичного аналізатора. Підсвічує помилки до того, як вони зламають гру.
    * **JavaScript (ES6) code snippets:** Для пришвидшення набору коду.
    * **Live Server:** Для роботи з чистим HTML. Дозволяє миттєво бачити зміни в браузері при збереженні файлу (гаряча перезавантаження).

### 💻 Terminal Survival Kit (Виживання в терміналі)

Ми працюємо як PRO — через термінал (в VS Code: `Ctrl + ~`).
Вам потрібно знати лише 4 команди навігації:

| Команда (Win/Mac) | Що робить | Приклад |
| :--- | :--- | :--- |
| `pwd` | **P**rint **W**orking **D**irectory. Де я зараз? | `pwd` -> `/Projects` |
| `ls` (або `dir`) | **L**i**s**t. Покажи файли в папці. | `ls` |
| `cd <назва>` | **C**hange **D**irectory. Зайти в папку. | `cd hive-mind` |
| `cd ..` | Вийти на рівень назад. | `cd ..` |

> **Лайфхак:** Почніть писати назву папки і натисніть `Tab`. Термінал сам допише назву.

## 2. The Build System: Node.js

У C++ у вас є `make`/`cmake`. У JS цю роль виконує **Node.js**.

1.  **Що це:**
    * **Node.js:** Runtime, що дозволяє запускати інструменти розробки (компілятори, лінтери).
    * **NPM:** Менеджер бібліотек (як `vcpkg`).
2.  **Скачати (LTS версію):** [nodejs.org](https://nodejs.org/)
3.  **Перевірка:** `node -v` та `npm -v`.

---

## 3. The Version Control: Git & GitHub

Інженер не носить код на флешці. Інженер використовує **Git**.

### Крок А: Інсталяція Git
1.  **Скачати:** [git-scm.com](https://git-scm.com/)
2.  **Конфігурація (в терміналі):**
    ```bash
    git config --global user.name "Ваше Ім'я"
    git config --global user.email "ваш@email.com"
    git config --global core.autocrlf true
    ```

### Крок Б: Паспорт розробника (GitHub Account)

Якщо у вас немає акаунту — створіть його зараз.
1.  Перейдіть на [github.com](https://github.com/).
2.  **Важливо:** Обирайте **професійний нікнейм** (напр., `vlad.dev`, `alex_code`). Цей профіль будуть бачити ваші майбутні роботодавці.
3.  *Порада:* Використовуйте пошту `.edu`, щоб отримати **GitHub Student Pack** (безкоштовні ліцензії).

### 🛑 Важливо: .gitignore

У корені проєкту створіть файл `.gitignore` і додайте туди:
```text
node_modules/
dist/
.env
.DS_Store

```

Це врятує ваш репозиторій від сміття.

## 5. Старт проєкту "Hive Mind"

Ми не починаємо з чистого аркуша. Ми розгортаємо інженерну заготовку курсового проєкту.

**Ваш алгоритм дій:**

1. **Fork (Виделка):**
* Зайдіть на [посилання репозиторію-стартера] (буде в LMS).
* Натисніть кнопку **Fork** (справа зверху). Тепер у вас є власна копія.


2. **Clone (Клонування):**
* На сторінці *вашого* форку натисніть `Code` -> скопіюйте посилання.
* У терміналі VS Code:


```bash
# 1. Завантажуємо код
git clone https://github.com/ВАШ_НІК/hive-mind-starter.git

# 2. ЗАХОДИМО В ПАПКУ (Критично важливо!)
cd hive-mind-starter

```


3. **Install & Run (Збірка і Запуск):**

```bash
# 1. Встановлюємо бібліотеки (аналог підключення .lib)
npm install

# 2. Запускаємо сервер розробки
npm run dev

```


*Натисніть `Ctrl + Click` по посиланню (Local: http://localhost:5173). Ви маєте побачити чорний екран гри.*

## Контрольні питання (Self-Check)

<details markdown="1">
<summary>1. Чому я маю писати `cd` після клонування?</summary>

**Відповідь:**
`git clone` створює нову папку поруч з вами. Ваш термінал залишається "ззовні". Щоб виконувати команди (`npm install`) всередині проєкту, ви повинні фізично зайти в цю папку.

</details>

<details markdown="1">
<summary>2. У Python є `venv`. Де моє віртуальне середовище в Node.js?</summary>

**Відповідь:**
У Node.js "venv" — це папка `node_modules` прямо в проєкті. Вона створюється командою `npm install` і містить всі бібліотеки локально.

</details>

<details markdown="1">
<summary>3. Яка різниця між Fork та Clone?</summary>

**Відповідь:**

* **Fork** — це дія на сервері (GitHub). Ви робите копію чужого репозиторію у свій акаунт.
* **Clone** — це дія на комп'ютері. Ви завантажуєте код з інтернету на диск.

</details>
