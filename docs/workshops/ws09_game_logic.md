# Воркшоп 9: Логіка гри та AI (мозок робота)

Ми маємо авторизацію. Ми маємо інтерфейс. Тепер нам потрібен гідний суперник.
У цьому воркшопі ми напишемо "мозок" для робота, який зможе навчатися на ходах гравця.

## 1. Архітектура Правил (Scalable Logic)

Новачки пишуть купу `if/else`:
*Якщо я Камінь і він Ножиці -> перемога...*

Інженер створює **Структуру Даних**.
Відкрийте `script.js` і створіть об'єкт правил. Це дозволить нам пізніше легко додати "Ящірку" і "Спока", не переписуючи логіку перевірки .

```javascript
const RULES = {
    'Rock': ['Scissors', 'Lizard'], // Що перемагає Камінь
    'Paper': ['Rock', 'Spock'],
    'Scissors': ['Paper', 'Lizard'],
    // ... легко розширювати
};

function getWinner(p1, p2) {
    if (p1 === p2) return 'draw';
    return RULES[p1].includes(p2) ? 'player' : 'computer';
}
```

## 2\. Рівні Інтелекту (AI Levels)

Ми реалізуємо патерн **Strategy**.

### Рівень 1: Deterministic (Дурний)

Робот завжди вибирає "Камінь".

  * **Задача:** Напишіть функцію, яка завжди повертає `index[0]`. Це потрібно для тестування вашої логіки перемоги .

### Рівень 2: Stochastic (Випадковий)

Використовуємо `Math.random()`.
Це класичний "чесний" рандом. Його неможливо передбачити, але він не намагається виграти .

### Рівень 3: Adaptive (Марковський ланцюг) 🧠

Це справжня інженерія. Робот має пам'ятати історію.

**Логіка:**

1.  Люди не рандомні. Якщо гравець вибрав "Камінь", є велика ймовірність, що наступним він вибере "Ножиці" (або навпаки).
2.  Ми створюємо матрицю ймовірностей:
    ```json
    {
      "Rock": { "Rock": 2, "Paper": 5, "Scissors": 1 }
    }
    ```
    *Це означає: після Каменя гравець 5 разів вибирав Папір.*
3.  **Завдання:** Напишіть функцію, яка дивиться на *попередній* хід гравця, знаходить найімовірніший *наступний* хід у матриці, і вибирає контр-зброю .

## 3\. Челендж: Super AI (GTO)

У теорії ігор існує поняття **Nash Equilibrium**. Для гри "Камінь-Ножиці-Папір" оптимальна стратегія (GTO) — це ідеальний рандом (33.3% на кожен варіант) .

  * **Питання:** Чому Адаптивний бот може програти професіоналу, а Рандомний бот — ніколи не буде в мінусі на довгій дистанції?

## 4\. Домашнє завдання

Реалізуйте перемикач складності у грі. Зробіть так, щоб Адаптивний бот починав як Рандомний, поки не набере достатньо статистики (мінімум 5 раундів), і тільки тоді вмикав "мозок".

## Контрольні питання

1. Чому структура даних `RULES = { Rock: ['Scissors', 'Lizard'], ... }` краще, ніж купа `if/else`?

<details markdown="1">
<summary>Відповідь</summary>

З `RULES` додавання нової зброї (Lizard, Spock) — це один рядок у словнику. Жодного рефакторингу логіки. З `if/else` — потрібно переписати або додати десятки умов. Структура даних = масштабованість.

</details>

2. Що таке Nash Equilibrium в контексті «Камінь-Ножиці-Папір»? Чому рандомний бот непереможний на довгій дистанції?

<details markdown="1">
<summary>Відповідь</summary>

Nash Equilibrium для цієї гри — рівномірно випадковий вибір (33.3% кожного варіанта). Жоден суперник не може отримати перевагу, якщо ви чисто рандомні. Адаптивний бот програє профі саме тому, що профі теж грає близько до рандому — немає патернів для навчання.

</details>

3. Що таке патерн Strategy? Як він реалізований в AI рівнях цього воркшопу?

<details markdown="1">
<summary>Відповідь</summary>

Strategy — об'єкт або функція, що замінює алгоритм на льоту без зміни коду, що його викликає. Тут: `function getAiMove() { return currentStrategy(); }`. `currentStrategy` — змінна, що вказує на `deterministicAI` / `randomAI` / `adaptiveAI`. Перемикач складності просто замінює цю змінну.

</details>

4. Чому Адаптивний бот (Марковський ланцюг) потребує мінімум 5 раундів перед активацією?

<details markdown="1">
<summary>Відповідь</summary>

З малою кількістю спостережень матриця переходів ненадійна — один-два ходи викривлюють розподіл. Наприклад, після 2 раундів `Rock→Rock: 2` виглядає як «гравець завжди повторює», хоча це може бути випадковість. 5+ раундів дають статистично більш стабільний базис для прийняття рішень.

</details>
