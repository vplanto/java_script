# Workshop: Hive Mind Architecture (Technical Brief)

> **Context:** Цей документ описує архітектуру Reference-імплементації, яку ми демонструємо на воркшопі. Ваша задача — відтворити цю логіку у Starter Kit.

## 1. Core Concept: Data-Oriented Design

Ми відмовляємось від класичного OOP (де клітинка — це об'єкт) на користь **Structure of Arrays (SoA)**. Це необхідно для продуктивності JS-двигуна (V8) при 10,000+ агентів.

### The Grid (Пам'ять)

Світ — це набір пласких масивів `Uint8Array` розміром `WIDTH * HEIGHT`. SoA застосовано саме до сітки; агенти в референсі зберігаються як масив об'єктів (кожен має `x`, `y`, `teamId`, `hasFood` тощо).

* **Static Layer:** `staticGrid[i]` — незмінні дані (0=Empty, 1=Wall, 2=Food). Індекс: `i = y * width + x`.
* **Dynamic Layer:** `pheromones[teamId][i]` — два окремі буфери (Blue=0, Red=1) для теплових карт феромонів.
* **Access:** Доступ через індекс `y * width + x`; межі перевіряються (поза сіткою — стіна / 0).

## 2. Agent Logic (Finite State Machine)

Мураха — це не "штучний інтелект", а простий автомат станів.

### States

1. **FORAGING (Пошук):**
* Рух: Random Walk + зважений вибір напрямку за градієнтом феромону *своєї* команди (якщо реалізовано).
* Дія: Якщо під агентом або в сусідній клітинці є їжа (`grid.hasFood`) → `grid.pickFood(x,y)` → перехід у стан **RETURNING**.
* Маркер: Залишає феромон "TO HOME" (максимальна інтенсивність).


2. **RETURNING (Повернення):**
* Рух: Векторна навігація до `homeX, homeY` (база).
* Дія: Якщо дистанція до бази &lt; 5px → скинути їжу → перехід у стан **FORAGING** і сигнал на спавн нового агента.
* Маркер: Можна залишати феромон "TO FOOD" (у референсі за замовчуванням лише FORAGING залишає "TO HOME").

### Sensing (Sensors)

Агент "сліпий". Він читає лише:

* Свою позицію `(x, y)`.
* Значення клітинки під собою та 8 сусідів.
* Вектор до ворога (тільки для Combat Phase).

### Spray (залишити феромон)

**Сенс:** феромон — це вказівка для *інших* мурах: «туди вже хтось рухався». Слід "TO HOME" означає шлях у бік бази (йшов з їжею); "TO FOOD" — у бік їжі. Інші агенти можуть використовувати градієнт цих значень для навігації (якщо реалізовано).

**Spray — не ресурс:** у референсі агент не має "запасу" феромону. Якщо за станом FSM він залишає слід (у FORAGING — "TO HOME"), то **кожен тік** записує у поточну клітинку `MAX_PHEROMONE`; це дія без обмежень. Обмежує лише випаровування вже записаного значення на сітці (decay).

## 3. The Rules of War (Combat System)

Бойова система детермінована і базується на **Lanchester's Laws** (Focus Fire).

**Підготовка (кожен тік після руху):**

1. **Detection:** Для кожного агента `A` рахуємо `EnemiesTargeting(A)` — скільки ворогів у радіусі атаки (`distSq <= 60`).
2. **Resolution:** Агент `A` **гине**, якщо в нього є хоч один ворог у радіусі **і** для якогось із цих ворогів `E`: `EnemiesTargeting(A) >= EnemiesTargeting(E)`.

**Усі можливі кейси (таблиця):**

| Сценарій | У агента A ворогів у радіусі | У ворога E ворогів у радіусі | Чи A гине? | Пояснення |
|----------|------------------------------|------------------------------|------------|-----------|
| Немає ворогів | 0 | — | Ні | Умова не виконується |
| **1v1** | 1 | 1 | **Так** | 1 ≥ 1 → гине (і A, і E) |
| **2v1** (A — той один) | 2 | 1 | **Так** | 2 ≥ 1 → одинак гине |
| **2v1** (A — один із двох) | 1 | 2 | Ні | 1 ≥ 2 — хибно → виживає |
| **2v2** | 2 | 2 | **Так** | 2 ≥ 2 → обидва боки гинуть |
| **3v1** (A — той один) | 3 | 1 | **Так** | 3 ≥ 1 → одинак гине |
| **3v1** (A — один із трьох) | 1 | 3 | Ні | 1 ≥ 3 — хибно → виживає |
| **N vs 1** (A — одинак) | N | 1 | **Так** | N ≥ 1 → гине |
| **N vs 1** (A — у групі) | 1 | N | Ні | 1 ≥ N при N>1 — хибно → виживає |

**Правило одним реченням:** Агент гине, якщо на нього нападає хтось **і** він сам «в меншості або на рівних» відносно хоча б одного з нападників (`EnemiesTargeting(A) >= EnemiesTargeting(E)`).

## 4. Simulation Loop (Game Tick)

Порядок виконання критичний для детермінізму:

1. **Випаровування феромонів (глобальний крок):**  
   Феромони з часом слабшають (як у реальних мурах). Для кожної клітинки **обох** шарів (Blue і Red):  
   - поточне значення множимо на `PHEROMONE_DECAY` (наприклад 0.99) — тобто кожен тік залишається 99% попереднього рівня;  
   - якщо після множення значення &lt; 1 — записати 0 (повне «випаровування»).  
   **Початкове значення:** коли агент залишає слід (Spray), у клітинку записується максимальна інтенсивність — `MAX_PHEROMONE` (наприклад 255). Далі воно з кожним тіком зменшується через decay.  
   Один прохід по всіх клітинках сітки, без участі агентів.

2. **Agent Update** — для кожного живого агента по черзі виконується один оновлювальний цикл. Див. флоу-граф нижче.

3. **Combat Resolution:** Після оновлення всіх агентів — визначити та видалити мертвих за таблицею правил бою (розділ 3).

4. **Render:** Малювання (не впливає на логіку).
