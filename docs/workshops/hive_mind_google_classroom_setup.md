# Завдання Google Classroom: Hive Mind — fork, npm, запуск бази

> **Призначення:** готовий текст для поля «Завдання» / «Інструкції» у Google Classroom (українською). Контекст курсу — лекція [JavaScript Runtime](../03_js_core.md), бриф [Hive Mind](../n00_HiveMind_Workshop.md), налаштування [Setup](../n01_setup.md).

## Назва завдання (рекомендовано)

**Hive Mind: fork репозиторію, `npm install`, запуск проєкту (dev-сервер)**

---

## Текст для студентів (вставити в Classroom)

### Мета

Підготувати **локальну робочу копію** курсового стартера **Hive Mind**: зробити **fork** на GitHub, **клонувати** собі на комп’ютер, встановити залежності (**npm**), **запустити базову збірку** і переконатися, що гра відкривається в браузері.

Доки код симуляції ви ще не дописуєте — важливо, щоб **проєкт стартував без помилок**.

---

### Передумови

- Встановлені **Node.js** (разом з **npm**) і **Git**. Перевірка в терміналі:

```bash
node -v
npm -v
git --version
```

Якщо чогось немає — скористайтесь інструкцією з налаштування середовища у матеріалах курсу (розділ про Node.js і Git).

---

### Крок 1. Fork (на GitHub)

1. Відкрийте репозиторій-стартер курсу (**посилання дає викладач у LMS / Classroom**, типово це форк від [hive-mind-starter](https://github.com/vplanto/hive-mind-starter)).
2. Натисніть **Fork** — з’явиться **ваша копія** репозиторію під **вашим** обліковим записом GitHub.

---

### Крок 2. Clone (на свій ПК)

1. На сторінці **вашого** форку натисніть **Code** і скопіюйте URL (HTTPS або SSH).
2. У терміналі (VS Code → Terminal):

```bash
git clone <URL_ВАШОГО_ФОРКУ>
cd hive-mind-starter
```

> **Важливо:** усі наступні команди (`npm install`, `npm run dev`) виконуються **вже всередині** папки проєкту, після `cd`.

---

### Крок 3. Встановити залежності (npm)

```bash
npm install
```

Це завантажує бібліотеки у папку `node_modules` (аналог «підключених пакетів» для збірки Vite).

---

### Крок 4. Запустити базу (dev-сервер)

```bash
npm run dev
```

У терміналі з’явиться посилання на локальний сервер (зазвичай **`http://localhost:5173`** для Vite). Відкрийте його в браузері (**Ctrl+клік** по посиланню або скопіюйте URL).

**Очікуваний результат:** сторінка проєкту відкривається; можливий «чорний екран» або заготовка гри — це нормально для старту, головне — **немає помилки в терміналі** і вкладка в браузері жива.

Зупинити сервер: у терміналі **Ctrl+C**.

---

### Що здати в Classroom

1. **Скріншот** браузера з відкритою сторінкою з **localhost** (видно адресну строку).
2. **Скріншот** терміналу, де видно успішний запуск (`npm run dev` і рядок з URL), **або** короткий текст: «Fork: `https://github.com/ВАШ_НІК/hive-mind-starter`» (посилання на ваш форк).

Якщо щось не запускається — опишіть текстом повідомлення помилки з терміналу **і** що саме ви вже зробили з кроків вище.

## Додатково для викладача (не копіювати студентам)

- Підставити в інструкцію **точне посилання** на стартер, якщо використовується форк організації, а не `vplanto/hive-mind-starter`.
- Нагадати про **WSL / шляхи**, якщо частина групи на Windows без WSL — `npm` має виконуватись у тій самій середовищі, де встановлений Node.
- Типові збої: забули `cd`; старий Node (бажано LTS); порт 5173 зайнятий — Vite запропонує інший порт, скріншот все одно валідний.
