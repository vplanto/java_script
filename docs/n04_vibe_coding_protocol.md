# Неймінг як Техніка Безпеки (Vibe Coding Protocol)

**Тип заняття:** Deep Dive / Safety Briefing
**Необхідний контекст:** Базовий JavaScript, досвід роботи з AI (ChatGPT/Claude)
**Мета:** Навчитись "демінеризувати" код, згенерований штучним інтелектом.

---

## 🤖 Вступ: AI — це п'яний джуніор

Ви вже почали використовувати AI для генерації коду ("Vibe Coding"). Це круто. Ви отримуєте результат швидко. Але є проблема.

AI не розуміє фізику вашого світу. Для LLM змінні `time`, `salary` та `distance` — це просто токени, які статистично часто зустрічаються поруч. Йому байдуже, що станеться, якщо ви додасте кілометри до доларів.

У C++ компілятор вдарить вас по руках, якщо ви спробуєте додати `std::string` до `int`.
У JavaScript (який ми вивчаємо) — **типізація динамічна**. Ви можете відняти "яблуко" від "поїзда", і JS видасть `NaN` (Not a Number) замість помилки, а ваш банківський додаток тихенько впаде в продакшні через тиждень.

**Неймінг в JS — це не естетика. Це єдина система безпеки, яка у вас залишилась.**

## 1. Правило "Магічних Одиниць" (Units of Measure)

Найдорожча помилка в історії IT сталася не через складний алгоритм, а через неймінг.
У 1999 році зонд NASA **Mars Climate Orbiter** вартістю $327 млн згорів в атмосфері Марса.
**Причина:** Одна команда інженерів передавала дані в *імперських одиницях* (фунти сили), а інша очікувала *метричні* (ньютони). У коді це виглядало як `force = 100`.

### Як це робить AI (Bad):

ШІ часто пише код, який виглядає "чистим", але є небезпечним:

```javascript
const timeout = 30; // 30 чого? Секунд? Хвилин? Тіків процесора?
const fileSize = 1024; // Байт? Кбіт? Мб?

```

Якщо ви передасте `30` у функцію, яка чекає мілісекунди (`setTimeout`), ваша програма чекатиме 0.03 секунди замість 30 секунд. Це баг, який важко знайти.

### Як це робите ви (Engineering Standard):

Ми додаємо суфікс одиниці виміру до будь-якої числової змінної, що має фізичний сенс.

```javascript
const timeout_ms = 30000;
const fileSize_bytes = 1024;
const price_usd = 99.99;

```

> **Правило:** Якщо змінна — це число, але не просто лічильник (`i`), вона зобов'язана мати суфікс.

## 2. Булеві Пастки (Boolean Traps)

ШІ обожнює називати прапорці (flags) іменниками. Це створює двозначність.

### Ситуація:

Ви читаєте код, згенерований Copilot:

```javascript
if (user.admin) {
    // ... grant access
}

```

**Що таке `user.admin`?**

1. Це об'єкт `AdminProfile`? (Тоді умова `true`, бо об'єкт існує).
2. Це рядок "SuperAdmin"? (Тоді `true`).
3. Це boolean `true/false`?

Якщо `user.admin` — це об'єкт, а ви чекаєте булеве значення, логіка може спрацювати помилково.

### Інженерний підхід:

Булева змінна повинна звучати як **питання**, на яке можна відповісти "Так" або "Ні".
Ми використовуємо префікси: `is`, `has`, `can`, `should`.

* ❌ `open = true` (Відкрито? Відкрити? Це команда?)
* ✅ `isOpen = true` (Стан).
* ❌ `write = false`
* ✅ `canWrite = false` (Права доступу).
* ❌ `user.valid`
* ✅ `user.isValid`

## 3. No Mystery Guests (Заборона x, y, data)

Коли ви просите AI написати обробку масиву, він видає це:

```javascript
// AI Code
const res = data.filter(x => x.a > 18).map(i => i.n);

```

Це працює. Але це "Write-Only Code". Ви не зможете це прочитати завтра.
У C++ ви бачите `Student* x`, і тип підказує вам, що це. У JS ви не бачите нічого.

### Refactoring Protocol:

Ваша задача як архітектора — замінити "сміттєві" імена на доменні сутності.

```javascript
// Human Refactor
const adultNames = students
    .filter(student => student.age > 18)
    .map(student => student.name);

```

**Чек-лист:**

1. Глобальний пошук по проєкту на слово `data`. Якщо це не шар абстракції бази даних — перейменувати.
2. Глобальний пошук змінних `info`, `item`, `obj`, `val`. Замінити на конкретну сутність (`product`, `message`, `user`).

## 🤔 Давайте поміркуємо: Code Review Game

Уявіть, що ви Tech Lead. AI-стажер приніс вам цей шматок коду для інтернет-магазину.
Знайдіть **3 потенційні проблеми** виключно на основі неймінгу.

```javascript
function processOrder(order) {
    if (order.status) {
        const total = order.items.reduce((a, b) => a + b.cost, 0);
        if (total > 1000) {
            applyDiscount(order, 10);
        }
    }
}

```

<details markdown="1">
<summary>🕵️ Розбір (натисніть для відповіді)</summary>

1. **`order.status` (Boolean Trap):** Незрозуміло, що перевіряється. Чи існує статус? Чи статус == "Active"? Якщо статус буде рядком "Pending", умова спрацює як `true`, що може бути помилкою.
* *Fix:* `order.status === 'COMPLETED'` або `order.isProcessed`.


2. **`b.cost` та `total` (Units Error):** В яких одиницях `cost`? Це копійки (cents) чи долари? У фінансових системах float (10.99) дає похибки округлення.
* *Fix:* `item.price_cents` та `total_cents`.


3. **`applyDiscount(order, 10)` (Magic Number):** 10 — це відсотки (10%) чи фіксована сума ($10)?
* *Fix:* Перейменувати параметр функції на `applyDiscountPercentage` або передавати `discount_amount_usd`.



**Висновок:** Цей код небезпечно пускати в продакшн, хоча синтаксично він вірний.

</details>

## ✅ Vibe Coding Checklist (Домашнє завдання)

Перед тим, як здавати будь-яку лабораторну роботу, де ви використовували AI, проженіть код через цей фільтр. Я буду відхиляти PR, якщо побачу порушення.

1. [ ] **Time & Money:** Усі змінні часу мають суфікси `_ms`, `_sec`. Гроші мають валюту `_usd`, `_uah`.
2. [ ] **Booleans:** Усі прапорці починаються з `is`, `has`, `can`.
3. [ ] **No "Data":** У коді немає змінних з назвою `data`, `info`, `res` (окрім низькорівневих бібліотек).
4. [ ] **No "x":** У циклах `forEach`/`map` ітератор має повну назву (`user` замість `u`).

**Пам'ятайте:** Код пишеться для людей, а не для машин. Машині все одно, а вам це ще дебажити о 3-й ночі.

## Контрольні питання

1. Яка помилка у цьому коді? Як виправити?

```javascript
const delay = 30;
setTimeout(callback, delay);
```

<details>
<summary>Відповідь</summary>

`delay = 30` — чого? Секунд? Хвилин? `setTimeout` очікує мілісекунди. Якщо ви мали на увазі 30 секунд, то `delay = 30` викличе callback через 30мс, а не 30 секунд.

Правильно: `const delay_ms = 30000;`

</details>

2. Чому `user.admin` є потенційно небезпечним прапорцем? Як правильно його назвати?

<details>
<summary>Відповідь</summary>

`user.admin` неоднозначний: це об'єкт AdminProfile? Рядок ролі? Boolean? `if (user.admin)` спрацює для будь-якого truthy значення.

Правильно: `user.isAdmin` (boolean) або `user.role === 'ADMIN'` (enum-порівняння).

</details>

3. AI написав `const res = data.filter(x => x.a > 18).map(i => i.n)`. Що не так з цим кодом?

<details>
<summary>Відповідь</summary>

`data`, `res`, `x`, `i`, `x.a`, `i.n` — «сміттєві» імена без доменного змісту. Читач не розуміє, що фільтрується і що повертається.

Правильно:
```javascript
const adultNames = students
  .filter(student => student.age > 18)
  .map(student => student.name);
```

</details>

4. Чому правило «Boolean змінна має відповідати на питання» — це не лише стиль, а безпека?

<details>
<summary>Відповідь</summary>

JavaScript — динамічно типізований. `if (user.status)` поверне `true` і для рядка `"Pending"`, і для об'єкта. Умова спрацює там, де не повинна.

`if (user.isActive === true)` або `if (user.status === 'ACTIVE')` виключають цю двозначність. В JS неймінг — єдиний документ, який описує очікуваний тип.

</details>
