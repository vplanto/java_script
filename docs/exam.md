### Екзаменаційний пул питань з Веб-інженерії

**Тема 0: Філософія інженерії та підхід до коду**
1. У чому полягає фундаментальна різниця між "кодером" та "інженером" у контексті використання штучного інтелекту (LLM) для написання коду?
2. Чому знання C++ та розуміння того, як працює пам'ять (Stack/Heap), є "суперсилою" для веб-інженера при роботі з двигуном браузера?
3. Поясніть концепцію "Міраж хибної майстерності" (Mirage of False Mastery) та пастку "Контекстного вікна" при роботі зі штучним інтелектом.

**Тема 1: HTML, DOM та Семантика**
1. Чому HTML класифікується як декларативна мова, і чим цей підхід відрізняється від імперативного (наприклад, у C++ чи JS)?
2. Поясніть концепцію браузера як "товстого клієнта". Чому зміна HTML-коду (наприклад, балансу рахунку) у DevTools локально не впливає на базу даних сервера?
3. Яка різниця між HTML4/XHTML та сучасним HTML5 (Living Standard) з точки зору філософії "Fault Tolerance"?
4. Чому використання несемантичної верстки ("Div Soup", наприклад `<div>` замість `<button>`) є інженерним антипатерном?

**Тема 2: Архітектура браузера та Critical Rendering Path**
1. З яких ізольованих процесів складається сучасний браузер (наприклад, Chrome) і чому архітектори обрали багатопроцесну модель замість однопроцесної?
2. Опишіть усі етапи Critical Rendering Path від розбору HTML до появи пікселів на екрані (DOM -> CSSOM -> Render Tree -> Layout -> Paint -> Composite).
3. Який етап рендерингу (Layout чи Composite) є "найдорожчим" для процесора? Поясніть, чому для плавних анімацій слід змінювати властивість `transform`, а не `width` чи `left`.
4. Чому елементи зі стилем `display: none` не потрапляють до Render Tree, тоді як елементи з `visibility: hidden` — потрапляють?
5. Для чого використовується властивість `z-index` і як на її поведінку впливає "Контекст накладання" (Stacking Context)?

**Тема 3: CSS як система координат (Box Model та Layout)**
1. У чому полягає критична проблема математики інтерфейсів при стандартному `box-sizing: content-box` і як її вирішує `box-sizing: border-box`?
2. Поясніть інженерну різницю між властивостями відступів: `padding` та `margin`. Що таке "Margin Collapse" (злиття відступів)?
3. Відносно чого обчислюються координати елемента при `position: absolute`, і чим це відрізняється від `position: fixed`?
4. Чому для розкладки елементів на сторінці використання CSS Grid є кращою практикою, ніж програмне обчислення та зміна координат через JavaScript?
5. Як працює каскад та специфічність селекторів у CSS (тег, клас, ID, !important)?

**Тема 4: JavaScript Core, V8 та Робота з пам'яттю**
1. Поясніть різницю між зберіганням змінних у Стек (Stack) та Купі (Heap). Де розміщуються примітиви, а де об'єкти?
2. У чому полягає "Пастка посилань" (Reference Trap) в JavaScript? Чому оператор `const` не захищає вміст масиву чи об'єкта від змін?
3. Чому копіювання через spread-оператор (`{ ...obj }`) створює лише поверхневу копію (shallow copy), і чим це загрожує для вкладених об'єктів?
4. Як працює збирач сміття (Garbage Collector) за алгоритмом Mark-and-Sweep, і як у JS може виникнути витік пам'яті (Memory Leak)?
5. Згідно з Vibe Coding Protocol, як інженер повинен іменувати булеві змінні (прапорці) та числові змінні, що мають фізичний сенс (наприклад, гроші чи час), для уникнення багів?

**Тема 5: Event Loop, Асинхронність та Web Workers**
1. Як працює Event Loop (Цикл подій) в JavaScript? Детально поясніть, що відбудеться, якщо викликати `setTimeout(fn, 0)`.
2. Чому нескінченний цикл (наприклад `while(true)`) "намертво" блокує інтерфейс користувача у браузері?
3. У чому полягає фундаментальна різниця між `setInterval` та `requestAnimationFrame`? Чому другий є галузевим стандартом для ігрових циклів та анімацій?
4. Що таке Web Workers і як вони допомагають розвантажити Main Thread при важких математичних обчисленнях (наприклад, при методі Монте-Карло)?
5. На відміну від потоків у C++ (std::thread), Web Workers у JS не мають спільної пам'яті. Як відбувається обмін даними між ними і чому в JS немає стану гонитви (Data Races)?

**Тема 6: JSON, Зберігання даних та Аналітика**
1. Що таке формат JSON? Які існують суворі правила синтаксису (лапки, коми) та які типи даних (наприклад, функції) категорично не підтримуються?
2. Чому виклик десеріалізації `JSON.parse()` завжди потрібно обгортати у блок `try/catch`?
3. Порівняйте JSON із XML: з яких інженерних причин JSON став сучасним стандартом передачі даних?
4. Яке головне обмеження має `localStorage` щодо типів даних, і який алгоритм необхідний для збереження і відновлення складного об'єкта гри?
5. Поясніть концепцію Blob (Binary Large Object). Як згенерувати JSON/XML файл для скачування безпосередньо в браузері?

**Тема 7: Мережева інженерія (OSI, TCP/IP, DNS)**
1. Поясніть різницю між протоколами транспортного рівня TCP та UDP. Чому онлайн-ігри (як CS:GO) використовують UDP, а не надійний TCP?
2. Як працює system доменних імен (DNS)? Опишіть повний ланцюжок від кешу браузера до DNS-серверів провайдера.
3. На якому рівні моделі OSI працює протокол HTTP, а на якому — IP-адресація? Чому рівні ізольовані один від одного?
4. Для чого у мережевих запитах використовується Порт (наприклад, :80, :443), якщо пристрій вже ідентифіковано за IP-адресою?
5. Поясніть, що таке пакет, і чому великі файли не передаються через мережу одним цілісним шматком.

**Тема 8: HTTP, REST API та Fetch**
1. Опишіть анатомію HTTP-запиту (Request) та HTTP-відповіді (Response). Де знаходяться заголовки (Headers) та тіло (Body)?
2. Яка різниця між HTTP-методами GET та POST? Чому форми з паролями ніколи не відправляють через GET?
3. Поясніть різницю між кодами помилок 401 (Unauthorized) та 403 (Forbidden).
4. Що таке ідемпотентність у REST API? Наведіть приклади ідемпотентних (наприклад, PUT) та неідемпотентних (POST) методів.
5. Чому при використанні `fetch()` we повинні явно перевіряти `response.ok`, замість того, щоб покладатися виключно на блок `catch` при 404 помилках?

**Тема 9: Веб-безпека (Security & Auth)**
1. У чому полягає різниця між атакою XSS (Cross-Site Scripting) та CSRF (Cross-Site Request Forgery)? Які методи захисту застосовуються для кожної?
2. Поясніть, чому використання `element.innerHTML = userInput` відкриває критичну вразливість для XSS, і чому `textContent` є безпечною альтернативою.
3. Як працює SQL Injection (ін'єкція)? Поясніть механізм захисту через Prepared Statements (параметризовані запити).
4. Чому хешування паролів за допомогою Web Crypto API на стороні клієнта (Front-end) вважається критичною помилкою в Production? Що таке Pass-the-Hash Attack?
5. Що таке Rainbow Table Attack і як використання "солі" (Salt) на сервері захищає базу даних паролів від зламу?

**Тема 10: Ігрові алгоритми та Архітектура (Game Dev)**
1. Чому для симуляції 10,000+ агентів (Hive Mind) використовується Data-Oriented Design та Structure of Arrays (SoA) замість класичного ООП (об'єкт "клітинка" чи "агент")?
2. Опишіть алгоритм детекції зіткнень AABB (Axis-Aligned Bounding Box), який використовується у грі Ping Pong. Які його обмеження?
3. Чим поведінка штучного інтелекту "God Mode" (телепортація) принципово відрізняється від "Predictive" алгоритму (математичного передбачення) у грі Pong?
4. Яку фундаментальну інженерну проблему може викликати алгоритм заливки Flood Fill, реалізований через рекурсію, на дуже великих полях у грі Minesweeper? Як це виправити?
5. У грі "Камінь-Ножиці-Папір" існує поняття Nash Equilibrium. Чому складний "адаптивний" бот на базі Марковських ланцюгів може програвати професіоналу, в той час як бот із чистим "рандомом" ніколи не буде у статистичному мінусі?
